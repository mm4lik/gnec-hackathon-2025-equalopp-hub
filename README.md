## ðŸ” Security Considerations 

* **Passwords**:
  Encrypted using **bcrypt hashing** with a strong salt factor, protecting against leaks even if the database is compromised.

* **API Protection**:
  Routes are secured with **JWT authentication** â€” users get a token after login that must be presented for protected actions.

* **Input Sanitization**:
  All user inputs (quiz answers, scenario choices, etc.) go through sanitization filters to prevent **XSS** (Cross-site scripting) and **NoSQL injection** attacks.

* **CORS Policy**:
  Configured to accept requests **only from trusted frontend domains** to prevent cross-site hijacking.

* **Rate Limiting**:
  API endpoints are guarded with request caps (e.g., 100 requests per hour per user) to prevent brute force attacks and bot abuse.
